# ТЗ — Система учёта ИИ-подписок в IT-компании

Версия 1 (черновик). Составлено Claude на основе документации 1С:Элемент.
Пользователь правит и дополняет — это рабочий документ, не финал.

---

## 1. Зачем это

IT-компания тратит деньги на разные ИИ-сервисы (ChatGPT, Claude, Copilot, Midjourney,
Gemini и т.д.). Подписки оформлены разными сотрудниками, часто с личных карт, по разным
тарифам и валютам. Никто не видит общей картины.

**Цель системы** — в одном месте видеть:
- какие ИИ-сервисы и подписки оплачивает компания;
- кто отвечает за подписку и на чей бюджет (отдел) она идёт;
- сколько это стоит в **месяц** и в **год** (прогноз расходов / run-rate);
- историю фактических оплат;
- что скоро спишется (контроль автопродления, чтобы не платить за ненужное).

**Кто пользуется:** финансовый менеджер / руководитель отдела / администратор IT.

## 2. Границы (что НЕ делаем в первой версии)

- Без автоматической интеграции с банком/биллингом сервисов — данные вводятся вручную.
- Без оплаты/эквайринга внутри приложения.
- Без сложных прав доступа на старте (добавим позже).
- Мультивалютность — по минимуму (храним валюту и сумму; пересчёт в рубли — простой курс,
  вводимый вручную; автокурс ЦБ — отдельная задача на потом).

## 3. Модель данных

Понятия платформы: **Справочник** = сущность/список; **реквизит** = поле;
**Перечисление** = фиксированный список; **Документ** = факт операции с датой.

### 3.1 Справочники

**Сервисы** — каталог ИИ-сервисов.
| Реквизит | Тип | Заметки |
|----------|-----|---------|
| Наименование | Строка | напр. «ChatGPT», «Claude» |
| Поставщик | Строка | напр. «OpenAI», «Anthropic» |
| Категория | Перечисление `КатегорияСервиса` | LLM / Изображения / Видео / Код / Аудио / Прочее |
| СайтURL | Строка | ссылка на сервис |
| Комментарий | Строка (многострочная) | |

**Отделы** — подразделения компании (на чей бюджет идут подписки).
| Реквизит | Тип |
|----------|-----|
| Наименование | Строка |
| Руководитель | `Сотрудники.Ссылка \| Неопределено` |

**Сотрудники** — кто владеет/оплачивает подписки.
| Реквизит | Тип |
|----------|-----|
| Наименование | Строка (ФИО) |
| Отдел | `Отделы.Ссылка \| Неопределено` |
| Должность | Строка |
| Email | Строка |

**Подписки** — центральная сущность. Одна запись = одна подписка на сервис.
| Реквизит | Тип | Заметки |
|----------|-----|---------|
| Наименование | Строка | напр. «ChatGPT Team — отдел разработки» |
| Сервис | `Сервисы.Ссылка` | |
| Тариф | Строка | напр. «Team», «Pro», «Enterprise» |
| Стоимость | Число | за один цикл оплаты |
| Валюта | Перечисление `Валюта` | USD / EUR / RUB |
| ЦиклОплаты | Перечисление `ЦиклОплаты` | Месячная / Годовая / Разовая / ПоИспользованию |
| КоличествоЛицензий | Число | seats; по умолчанию 1 |
| ДатаНачала | Дата | |
| ДатаСледующегоСписания | Дата | для отчёта «предстоящие списания» |
| Статус | Перечисление `СтатусПодписки` | Активна / Триал / Приостановлена / Отменена |
| Автопродление | Булево | |
| Ответственный | `Сотрудники.Ссылка` | кто отвечает |
| Отдел | `Отделы.Ссылка` | на чей бюджет |
| Комментарий | Строка (многострочная) | |

> Расчётное: **МесячнаяСтоимость** = Стоимость, приведённая к месяцу
> (Годовая → /12, Разовая → 0 в run-rate). Считается в коде/отчёте, не хранится.

### 3.2 Документы

**Оплата** — факт списания денег по подписке.
| Реквизит | Тип | Заметки |
|----------|-----|---------|
| Дата | Дата | дата списания |
| Подписка | `Подписки.Ссылка` | |
| Сумма | Число | |
| Валюта | Перечисление `Валюта` | |
| СуммаВРублях | Число | пересчёт по курсу (вручную на старте) |
| СпособОплаты | Перечисление `СпособОплаты` | КорпКарта / ЛичнаяКартаВозмещение / Счёт |
| Комментарий | Строка | |

### 3.3 Перечисления

- **КатегорияСервиса**: LLM, Изображения, Видео, Код, Аудио, Прочее.
- **ЦиклОплаты**: Месячная (по умолчанию), Годовая, Разовая, ПоИспользованию.
- **СтатусПодписки**: Активна (по умолчанию), Триал, Приостановлена, Отменена.
- **СпособОплаты**: КорпКарта (по умолчанию), ЛичнаяКартаВозмещение, Счёт.
- **Валюта**: RUB, USD, EUR. *(Если нужна гибкость — позже сделать справочником с курсом.)*

## 4. Экраны (формы)

Для каждого справочника — `ФормаСписка` (таблица) и `ФормаОбъекта` (карточка).

- **Подписки — список:** колонки Наименование, Сервис, Стоимость+Валюта, ЦиклОплаты,
  Статус (значком с цветом, как приоритет в примере page_005), Отдел, Ответственный,
  ДатаСледующегоСписания. Фильтры по Статусу, Отделу, Сервису.
- **Подписки — карточка:** все реквизиты; внизу — список связанных оплат.
- **Сервисы / Сотрудники / Отделы — списки и карточки** (простые).
- **Оплаты — список:** Дата, Подписка, Сумма+Валюта, СуммаВРублях, СпособОплаты.
- **Командный интерфейс:** разделы навигации — Подписки, Оплаты, Сервисы, Сотрудники,
  Отделы, Отчёты.

## 5. Отчёты

1. **Расходы по месяцам** — сумма оплат по месяцам (диаграмма + таблица). Главный отчёт.
2. **Расходы по сервисам** — топ ИИ-сервисов по затратам за период.
3. **Расходы по отделам** — сколько тратит каждый отдел.
4. **Активные подписки и прогноз** — список активных подписок + суммарный
   месячный и годовой run-rate (во что обходится «как сейчас»).
5. **Предстоящие списания** — подписки с ДатаСледующегоСписания в ближайшие 30 дней
   (контроль автопродления).

## 6. Этапы разработки (Definition of Done)

**Этап 0 — связка (нужен пользователь):**
создать приложение в 1С, включить режим разработки, подключить наш GitHub-репозиторий,
сделать первую синхронизацию, прислать формат файлов. (См. workflow.md, шаг 0.)

**Этап 1 — MVP-данные:**
справочники Сервисы, Отделы, Сотрудники, Подписки + все перечисления; документ Оплата;
автоматические формы. Можно вручную внести данные и они сохраняются.

**Этап 2 — интерфейс:**
настроенные ФормаСписка/ФормаОбъекта, цветной статус подписки, командный интерфейс
с разделами, в карточке подписки виден список её оплат.

**Этап 3 — отчёты:**
отчёты 1–5 из раздела 5. Главный — «Расходы по месяцам».

**Этап 4 — удобство (после MVP):**
прогноз/run-rate на дашборде, напоминания о предстоящих списаниях, простой пересчёт валют,
права доступа по ролям, выгрузка в Excel.

## 7. Открытые вопросы к пользователю

1. Сколько примерно сотрудников/отделов/подписок? (влияет на формы и отчёты)
2. Нужна ли мультивалютность сейчас или достаточно рублей?
3. Подписки оплачивают с корпоративной карты, личных карт с возмещением, или и так и так?
4. Нужны ли роли (кто что видит/редактирует) уже в первой версии?
5. Нужна ли выгрузка отчётов в Excel/печать?
