# Как мы вместе ведём разработку

Платформа 1С:Элемент облачная, поэтому работа идёт в паре:
**Claude пишет файлы → ты переносишь их в среду разработки 1С и проверяешь.**

## Роли

- **Claude Code** — пишет и правит файлы проекта в этом репозитории, проектирует данные,
  пишет код на языке «1С:Элемент», отчёты, инструкции. Коммитит и пушит в GitHub.
- **Ты** — «руки» в облаке 1С: импортируешь/синхронизируешь проект в среде разработки,
  запускаешь, тестируешь, присылаешь Claude результат (ошибки, скриншоты, поведение).

## Цикл одной задачи

1. **Claude** делает изменение в репозитории (новый справочник, поле, отчёт…) и пушит в GitHub.
2. **Ты** обновляешь проект в 1С из Git (импорт/синхронизация ветки).
3. **Ты** публикуешь и запускаешь приложение в облаке (Ctrl+F5 → «Опубликовать и запустить»).
4. **Ты** проверяешь и пишешь Claude: что получилось / что сломалось (текст, ошибка, скриншот).
5. **Claude** правит и снова пушит. Повтор, пока задача не закрыта.

## Шаг 0 (делаем один раз, в начале) — связать 1С и GitHub

Нужно от тебя, по документации платформы:

1. Создать новый проект/приложение в панели управления 1С, включить **режим разработки**
   (руководство: page_019, page_005).
2. Подключить **внешний Git-репозиторий** — наш GitHub-репозиторий этого проекта
   (page_042 «Как подключить внешний Git-репозиторий»).
3. Сделать первую синхронизацию и **прислать Claude**, как выглядит выгрузка проекта в Git
   (какие появились файлы и папки). Это нужно, чтобы Claude писал файлы в **точно том**
   формате, который понимает 1С.

> Пока шаг 0 не сделан, Claude работает «вслепую по документации»: проектирует данные и
> пишет ТЗ/черновики кода. Реальный формат файлов подтверждаем после первой синхронизации.

## Что присылать Claude при проблемах

- Текст ошибки целиком (скопировать), а не пересказ.
- Скриншот экрана среды разработки или приложения.
- Что именно нажал и что ожидал увидеть.
