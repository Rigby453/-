# Платформа 1С:Предприятие.Элемент — что важно знать

Краткая выжимка из официального руководства (папка `ManualDevelop`), на которую
опирается весь проект. Источник по каждому пункту указан как `page_XXX`.

## Что это за платформа

«1С:Предприятие.Элемент» — **облачная low-code платформа** для веб-кабинетов,
порталов, браузерных и мобильных приложений (page_001).

- Проект и работающее приложение хранятся **на сервере 1С** (в облаке).
- Разработчик создаёт и отлаживает проект в **веб-среде разработки** (в браузере), page_002.
- Язык — **«1С:Элемент»** (русские ключевые слова, на основе языка «1С:Предприятие 8»).
- Есть **язык запросов**, готовые **UI-компоненты**, **отчёты и аналитика**.

## Ключевые понятия (page_003)

- **Проект** — полностью описывает логику приложения. Имеет имя и версии (версия 1, 2, …).
- **Приложение** — исполняется сервером; = версия проекта + СУБД (данные) + хранилище файлов.
- **Сервер** — исполняет приложения. **Пользователи** работают через веб/мобильный клиент.
- **Панель управления** — администрирование (создать приложение, пользователей, включить
  режим разработки).
- **Среда разработки** — где создаётся проект.

## Из чего собирается проект (виды элементов, page_005, page_035)

- **Подсистема** — группировка элементов проекта (напр. «Основное»).
- **Справочник** — хранит списки/сущности (напр. Задачи, Проекты). Имеет **реквизиты** (поля).
  По умолчанию есть реквизит **Наименование**; можно добавить стандартный **Код**.
- **Перечисление** — фиксированный список значений (напр. ПриоритетЗадачи: Высокий/Средний/Низкий),
  можно задать значение **ПоУмолчанию**.
- **Документ** — регистрирует факт/операцию (напр. оплату) с датой.
- **Форма** — `ФормаСписка` (таблица) и `ФормаОбъекта` (карточка записи). Может генерироваться
  автоматически (**автоматический интерфейс**) или настраиваться вручную в редакторе компонентов.
- **Отчёт** — таблицы и диаграммы по данным (page_006).
- **Модуль** — код на языке «1С:Элемент». Методы могут иметь аннотации видимости
  (напр. `@ВПодсистеме`).

Типы реквизитов: `Строка`, `Число`, `Булево`, `Дата`/`Момент`, ссылка на справочник
(`Справочник.Ссылка | Неопределено`), перечисление и др.

### Пример кода на языке «1С:Элемент» (page_005)

```
@ВПодсистеме
метод ПолучитьЦвет(): Цвет
    выбор этот
    когда Высокий
        возврат Цвета.Красный
    когда Средний
        возврат Цвета.Желтый
    когда Низкий
        возврат Цвета.Зеленый
    иначе
        возврат Цвета.Зеленый
    ;;
```

## Git и совместная работа (page_038–043, 052)

- Для групповой разработки штатно используется **GitLab** на сервере 1С.
- Можно подключить **внешний Git-репозиторий** (page_042) и импортировать проект
  из Git (page_043) — но при «просто Git» доступны только основные функции.
- Файл проекта можно загрузить в ветку (page_052), переключаться между ветками (page_053).

**Вывод для нас:** проект 1С представим в виде файлов в Git-репозитории. Значит, Claude
может править эти файлы, а пользователь — синхронизировать их со средой разработки 1С.

## Граница ответственности Claude Code

- ✅ Может: читать документацию, писать/править файлы проекта в репозитории, проектировать
  данные, писать код на языке «1С:Элемент», тексты, отчёты, инструкции.
- ❌ Не может: войти в облачную среду разработки 1С, «нажимать кнопки», запускать и
  публиковать приложение, видеть его экран. Это делает пользователь (см. workflow.md).

> Открытый вопрос для проверки на практике: точный формат выгрузки проекта во внешний Git
> (имена файлов/папок, кодировка). Уточняем при первой синхронизации — см. workflow.md, шаг 0.
